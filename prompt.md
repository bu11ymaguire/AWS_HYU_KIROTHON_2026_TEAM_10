한양대 서울캠퍼스 시간표 추천 챗봇을 만들고자 합니다.

아래 요구사항으로 Spec을 만들어주세요.

1/ 참조 데이터 구조화

   - #data.md 에 담긴 학사안내 원문을 챗봇이 참조할 수 있도록 

     구조화된 형태(JSON/YAML 등)로 파싱

   - 최소한 다음 영역을 분리: 수강신청 일정, 학점 제한 규칙, 

     폐강 기준, 선수-후수 관계, 동일/대치 교과목, 교육과정 적용 규칙

2/ 시간표 추천 로직

   - 사용자 입력: 학번, 학년, 소속 학과, 이수 현황, 희망 과목 리스트

   - 학점 상한/하한 자동 체크 (학년별, 졸업예정 여부, 학업연장 여부에 따라 다름)

   - 시간 충돌 검증

   - 선수-후수 과목 미이수 시 경고

   - 폐강 위험 과목 표시 (수강인원 기준)

3/ 챗봇 인터페이스

   - 자연어로 질문하면 학사 규정 기반으로 답변

     (예: "나 3학년인데 최대 몇 학점까지 들을 수 있어?", 

          "기초학술영어 안 들었는데 전문학술영어 신청 가능해?")

   - 추천 시간표를 표 형태로 출력

4/ RAG 파이프라인

   - data.md를 청크 단위로 분할 → 임베딩 → 벡터 DB 저장

   - 사용자 질문에 대해 관련 청크를 검색 후 LLM에 컨텍스트로 전달

   - 환각 방지를 위해 출처(페이지 번호) 함께 표시

5/ 로컬 개발 환경

   - Python 기반, LangChain 또는 LlamaIndex 활용

   - 벡터 DB는 로컬에서 Chroma 또는 FAISS 사용

   - .env로 API 키 등 환경변수 관리

   - CLI 모드로 로컬 테스트 가능해야 함

6/ 데이터 전처리

   - data.md의 표 형식 데이터(학사일정, 학점 기준표 등)가 

     현재 깨져 있으므로 정규화 스크립트 포함

   - 페이지 구분자(--- 페이지 N ---)를 메타데이터로 활용
