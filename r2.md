# 요구사항 문서

## 소개

한양대 수강신청 AI Agent는 학생이 졸업사정 이미지를 업로드하고 기본 정보(이름, 전공, 학년, 교양 관심분야(선택))를 입력하면, Amazon Bedrock Claude Vision으로 졸업 현황을 분석하고 Claude Sonnet으로 최적의 강의 3개를 추천해주는 서비스입니다. 에브리타임 강의평 데이터를 기반으로 추천 이유와 평점을 함께 제공하며, 학사 규정 Q&A(RAG 기반), 시간표 충돌 검증, Intent 기반 자동 분류 기능을 통해 학생이 수강신청 결정에 필요한 모든 정보를 빠르고 정확하게 얻을 수 있도록 돕습니다. 결과는 전공 추천, 교양 추천 두가지로 출력.

---

## 용어 정의 (Glossary)

- **Agent**: 한양대 수강신청 AI Agent 시스템 전체
- **Vision_Analyzer**: Amazon Bedrock Claude Vision 기반 졸업사정 이미지 분석 모듈
- **Course_Recommender**: Amazon Bedrock Claude Sonnet 기반 강의 추천 모듈
- **Course_DB**: DynamoDB에 저장된 에브리타임 크롤링 강의 데이터
- **Graduation_Status**: Vision_Analyzer가 이미지에서 추출한 졸업사정 분석 결과 객체
- **Student_Profile**: 학생이 입력한 전공, 학년, 관심분야 정보 객체
- **Recommendation_Result**: Course_Recommender가 생성한 추천 강의 3개와 추천 이유 객체
- **Dev_Agent**: 코드 작성을 담당하는 AI 서브에이전트
- **QA_Agent**: 테스트 코드 작성을 담당하는 AI 서브에이전트
- **Review_Agent**: 코드 리뷰를 담당하는 AI 서브에이전트
- **Infra_Agent**: AWS CDK로 인프라를 자동 생성하는 AI 서브에이전트
- **Regulation_QA**: Amazon Bedrock Claude Sonnet 기반 학사 규정 질의응답 모듈 (RAG 파이프라인)
- **Vector_DB**: Amazon OpenSearch Serverless 기반 벡터 데이터베이스
- **Embedding_Engine**: Amazon Bedrock Titan Embeddings 기반 임베딩 생성 모듈
- **Schedule_Validator**: 시간표 충돌 검증 모듈
- **Intent_Classifier**: 사용자 입력을 의도(Intent)별로 분류하는 모듈
- **Data_Pipeline**: courses.csv → DynamoDB, data.md → Vector_DB 데이터 적재 파이프라인
- **Regulation_Source**: 한양대 2026-1학기 학사안내 원문 (data/data.md)
- **Chunk**: Regulation_Source를 800자 단위로 분리한 텍스트 조각
- **Intent**: 사용자 요청의 의도 분류 결과 (GRADUATION, COURSE_RECOMMEND, SCHEDULE_INFO, CREDIT_CHECK, DROP_CHECK, PREREQUISITE, REGULATION_QA)

## 요구사항

### 요구사항 1: 학생 정보 입력

**User Story:** As a 한양대 재학생, I want to 전공, 학년, 교양 관심분야를 입력하고 졸업사정 이미지를 업로드하고 싶다, so that AI가 나의 상황에 맞는 강의를 전공, 교양 두가지로 추천해줄 수 있다.

#### 인수 기준

1. THE Agent SHALL 전공, 학년, 교양 관심분야 텍스트 입력 필드와 이미지 업로드 영역을 포함한 입력 폼을 제공한다.
2. WHEN 사용자가 졸업사정 이미지 파일을 업로드하면, THE Agent SHALL 해당 파일이 PNG, JPG, JPEG 형식인지 검증한다.
3. IF 업로드된 파일이 PNG, JPG, JPEG 이외의 형식이면, THEN THE Agent SHALL "지원하지 않는 파일 형식입니다. PNG, JPG, JPEG 파일을 업로드해주세요." 오류 메시지를 표시한다.
4. IF 업로드된 파일 크기가 10MB를 초과하면, THEN THE Agent SHALL "파일 크기는 10MB 이하여야 합니다." 오류 메시지를 표시한다.
5. IF 전공, 학년, 관심분야 중 하나라도 입력되지 않은 상태에서 분석을 요청하면, THEN THE Agent SHALL 미입력 필드를 강조 표시하고 입력을 요청하는 메시지를 표시한다. 단, 교양 관심분야는 선택사항으로 만든다. 
6. IF 졸업사정 이미지가 업로드되지 않은 상태에서 분석을 요청하면, THEN THE Agent SHALL "졸업사정 이미지를 업로드해주세요." 오류 메시지를 표시하지 않고, 졸업사정을 기반하지 않은 강의추천만 진행한다.

---

### 요구사항 2: 졸업사정 이미지 분석

**User Story:** As a 한양대 재학생, I want to 졸업사정 캡처 이미지를 업로드하면 자동으로 분석되길 원한다, so that 졸업까지 무엇이 부족한지 직접 계산하지 않아도 된다.

#### 인수 기준

1. WHEN 사용자가 유효한 졸업사정 이미지와 학생 정보를 제출하면, THE Vision_Analyzer SHALL Amazon Bedrock Claude Vision API를 호출하여 이미지를 분석한다.
2. THE Vision_Analyzer SHALL 이미지에서 이수 완료 과목 목록, 미이수 과목 목록, 현재 취득 학점, 졸업까지 남은 학점, 영역별 충족 여부를 추출하여 Graduation_Status 객체로 반환한다.
3. WHEN Vision_Analyzer가 이미지 분석을 완료하면, THE Agent SHALL Graduation_Status 객체를 Course_Recommender에 전달한다.
4. IF Amazon Bedrock Vision API 호출이 실패하면, THEN THE Vision_Analyzer SHALL "졸업사정 분석에 실패했습니다. 잠시 후 다시 시도해주세요." 오류 메시지를 반환한다.
5. IF 업로드된 이미지에서 졸업사정 정보를 추출할 수 없으면, THEN THE Vision_Analyzer SHALL "졸업사정 정보를 인식할 수 없습니다. 선명한 이미지를 업로드해주세요." 오류 메시지를 반환한다.
6. WHILE Vision_Analyzer가 이미지를 분석하는 동안, THE Agent SHALL 로딩 상태를 사용자에게 표시한다.

---

### 요구사항 3: 강의 데이터 조회

**User Story:** As a 한양대 재학생, I want to 에브리타임 강의평 데이터를 기반으로 추천을 받고 싶다, so that 직접 에브리타임을 검색하지 않아도 강의 평점과 후기를 확인할 수 있다.

#### 인수 기준

1. THE Course_DB SHALL 강의명, 교수명, 학점, 영역 구분, 에브리타임 평점, 강의평 요약 데이터를 저장한다.
2. WHEN Course_Recommender가 강의 추천을 시작하면, THE Agent SHALL Student_Profile의 전공과 Graduation_Status의 미이수 영역을 기준으로 Course_DB에서 후보 강의 목록을 조회한다.
3. THE Agent SHALL Course_DB 조회 시 에브리타임 평점 3.0 이상인 강의만 후보로 포함한다.
4. IF Course_DB 조회 결과가 0건이면, THEN THE Agent SHALL "조건에 맞는 강의 데이터가 없습니다." 메시지를 반환하고 추천 조건 완화를 안내한다.

---

### 요구사항 4: AI 강의 추천

**User Story:** As a 한양대 재학생, I want to 내 졸업사정과 관심분야를 고려한 강의 3개를 추천받고 싶다, 전공 3개, 교양 3개 각각 추천해준다. so that 수강신청 시간을 절약하고 최적의 강의를 선택할 수 있다.

#### 인수 기준

1. WHEN Graduation_Status와 Student_Profile이 모두 준비되면, THE Course_Recommender SHALL Amazon Bedrock Claude Sonnet API를 호출하여 최적 강의 3개를 추천한다.
2. THE Course_Recommender SHALL 졸업사정 분석 결과, 학생 전공, 학년, 관심분야, Course_DB 후보 강의 목록을 모두 포함한 프롬프트를 구성하여 Bedrock에 전달한다.
3. THE Course_Recommender SHALL 각 추천 강의에 대해 강의명, 교수명, 학점, 에브리타임 평점, 추천 이유(2문장 이내)를 포함한 Recommendation_Result를 반환한다.
4. IF Amazon Bedrock Sonnet API 호출이 실패하면, THEN THE Course_Recommender SHALL "강의 추천에 실패했습니다. 잠시 후 다시 시도해주세요." 오류 메시지를 반환한다.
5. WHILE Course_Recommender가 추천을 생성하는 동안, THE Agent SHALL 로딩 상태를 사용자에게 표시한다.
6. THE Course_Recommender SHALL 졸업 필수 미이수 과목을 관심분야 기반 선택과목보다 우선순위 높게 추천한다.

---

### 요구사항 5: 통합 결과 화면 표시

**User Story:** As a 한양대 재학생, I want to 분석 결과와 추천 강의를 한 화면에서 보고 싶다, so that 수강신청 결정에 필요한 모든 정보를 한눈에 파악할 수 있다.

#### 인수 기준

1. WHEN Recommendation_Result가 준비되면, THE Agent SHALL 추천 강의 3개를 카드 형태로 화면에 표시한다.
2. THE Agent SHALL 각 강의 카드에 강의명, 교수명, 학점, 에브리타임 평점, 추천 이유를 표시한다.
3. THE Agent SHALL 졸업까지 남은 학점을 영역별로 시각적 진행 바(progress bar) 형태로 표시한다.
4. THE Agent SHALL 이수 완료 영역은 초록색, 미이수 영역은 빨간색으로 구분하여 표시한다.
5. WHEN 사용자가 결과 화면에서 "다시 분석하기" 버튼을 클릭하면, THE Agent SHALL 입력 폼 화면으로 초기화하여 이동한다.

### 요구사항 6: Multi-Agent 개발 구조

**User Story:** As a 개발팀, I want to Dev/QA/Review Agent가 역할을 분담하여 자동으로 개발하길 원한다, so that 코드 품질을 유지하면서 빠르게 개발할 수 있다.

#### 인수 기준

1. THE Dev_Agent SHALL 기능 요구사항에 따라 React 컴포넌트, Lambda 핸들러, CDK 인프라 코드를 작성한다.
2. THE QA_Agent SHALL Dev_Agent가 작성한 코드에 대한 단위 테스트 및 통합 테스트 코드를 작성한다.
3. THE Review_Agent SHALL Dev_Agent가 작성한 코드를 검토하고 개선 사항을 피드백한다.
4. THE Infra_Agent SHALL AWS CDK를 사용하여 DynamoDB 테이블, Lambda 함수, API Gateway를 AWS 콘솔 직접 접근 없이 자동 생성한다.
5. WHEN 파일이 저장되면, THE Agent SHALL 자동으로 테스트를 실행한다.
6. WHEN 모든 테스트가 통과하면, THE Agent SHALL AWS Amplify를 통해 자동 배포를 트리거한다.

### 요구사항 7: 성능 및 보안

**User Story:** As a 한양대 재학생, I want to 빠르고 안전하게 서비스를 이용하고 싶다, so that 수강신청 기간에 서비스가 원활하게 동작한다.

#### 인수 기준

1. WHEN 사용자가 분석을 요청하면, THE Agent SHALL 전체 분석 및 추천 결과를 30초 이내에 반환한다.
2. THE Agent SHALL 업로드된 졸업사정 이미지를 서버에 영구 저장하지 않고 분석 완료 후 즉시 삭제한다.
3. THE Agent SHALL API Gateway에서 모든 요청에 대해 HTTPS를 강제 적용한다.
4. THE Agent SHALL Lambda 함수에서 환경 변수로 Bedrock API 키 및 DynamoDB 테이블명을 관리하고 소스코드에 하드코딩하지 않는다.
5. WHILE 동시 요청이 100건 이하인 동안, THE Agent SHALL 서비스 응답 시간을 30초 이내로 유지한다.
6. IF Lambda 함수 실행 시간이 29초를 초과하면, THEN THE Agent SHALL 타임아웃 오류를 반환하고 사용자에게 재시도를 안내한다.

### 요구사항 8: Intent 분류 시스템

**User Story:** As a 한양대 재학생, I want to 자유롭게 질문하면 시스템이 자동으로 의도를 파악하길 원한다, so that 별도의 메뉴 선택 없이 원하는 기능을 바로 이용할 수 있다.

#### 인수 기준

1. WHEN 사용자가 텍스트 입력 또는 이미지를 제출하면, THE Intent_Classifier SHALL 사용자 요청을 GRADUATION, COURSE_RECOMMEND, SCHEDULE_INFO, CREDIT_CHECK, DROP_CHECK, PREREQUISITE, REGULATION_QA 중 하나의 Intent로 분류한다.
2. WHEN Intent가 GRADUATION으로 분류되면, THE Agent SHALL 졸업사정 이미지 분석 흐름(Vision_Analyzer)을 실행한다.
3. WHEN Intent가 COURSE_RECOMMEND로 분류되면, THE Agent SHALL 강의 추천 흐름(Course_Recommender)을 실행한다.
4. WHEN Intent가 SCHEDULE_INFO, CREDIT_CHECK, DROP_CHECK, PREREQUISITE 중 하나로 분류되면, THE Agent SHALL 해당 Intent에 맞는 키워드를 포함하여 Regulation_QA 모듈로 질문을 전달한다.
5. WHEN Intent가 REGULATION_QA로 분류되면, THE Agent SHALL 사용자 질문을 그대로 Regulation_QA 모듈로 전달한다.
6. IF 사용자 입력에서 Intent를 판별할 수 없으면, THEN THE Intent_Classifier SHALL REGULATION_QA를 기본 Intent로 적용하여 RAG 폴백 처리한다.
7. THE Intent_Classifier SHALL Amazon Bedrock Claude Sonnet API를 사용하여 Intent를 분류한다.

---

### 요구사항 9: 학사 규정 Q&A (RAG)

**User Story:** As a 한양대 재학생, I want to 학사 규정에 대해 자유롭게 질문하면 정확한 답변을 받고 싶다, so that 학사안내 문서를 직접 찾아보지 않아도 된다.

#### 인수 기준

1. THE Regulation_QA SHALL Regulation_Source(data/data.md)를 800자 단위 Chunk로 분리하여 Vector_DB에 저장한다.
2. THE Embedding_Engine SHALL Amazon Bedrock Titan Embeddings API를 사용하여 각 Chunk의 임베딩 벡터를 생성한다.
3. WHEN 사용자가 학사 규정 관련 질문을 입력하면, THE Embedding_Engine SHALL 해당 질문의 임베딩 벡터를 생성한다.
4. WHEN 질문 임베딩이 생성되면, THE Regulation_QA SHALL Vector_DB에서 코사인 유사도 기준 상위 5개 Chunk를 검색한다.
5. WHEN 유사 Chunk 5개가 검색되면, THE Regulation_QA SHALL 검색된 Chunk와 사용자 질문을 Amazon Bedrock Claude Sonnet API에 전달하여 답변을 생성한다.
6. THE Regulation_QA SHALL 수강신청 일정, 학점 상한/하한 규칙, 선수-후수 과목 관계, 폐강 기준, 동일/대치 교과목, 수강포기 조건, 교육과정 적용 규칙에 대한 질문을 처리한다.
7. IF 검색된 Chunk에서 질문에 대한 근거를 찾을 수 없으면, THEN THE Regulation_QA SHALL "해당 정보를 찾을 수 없습니다." 응답을 반환한다.
8. THE Regulation_QA SHALL 답변 생성 시 근거가 된 Chunk의 원문 출처를 함께 표시한다.
9. IF Amazon Bedrock Titan Embeddings API 호출이 실패하면, THEN THE Regulation_QA SHALL "학사 규정 검색에 실패했습니다. 잠시 후 다시 시도해주세요." 오류 메시지를 반환한다.
10. IF Amazon Bedrock Sonnet API 호출이 실패하면, THEN THE Regulation_QA SHALL "답변 생성에 실패했습니다. 잠시 후 다시 시도해주세요." 오류 메시지를 반환한다.
11. FOR ALL 유효한 학사 규정 질문에 대해, 질문을 임베딩하고 Chunk를 검색한 후 답변을 생성하고 다시 동일 질문을 처리하면 동등한 답변을 반환한다 (라운드트립 일관성).
12. WHILE Regulation_QA가 답변을 생성하는 동안, THE Agent SHALL 로딩 상태를 사용자에게 표시한다.

---

### 요구사항 10: 시간표 충돌 검증

**User Story:** As a 한양대 재학생, I want to 희망 과목 목록을 입력하면 시간 충돌을 자동으로 확인하고 싶다, so that 수강신청 시 시간표 겹침으로 인한 실수를 방지할 수 있다.

#### 인수 기준

1. WHEN 사용자가 희망 과목 목록을 입력하면, THE Schedule_Validator SHALL 각 과목의 요일 및 교시 정보를 Course_DB에서 조회한다.
2. WHEN 과목 시간 정보가 조회되면, THE Schedule_Validator SHALL 모든 과목 쌍에 대해 시간 충돌 여부를 검사한다.
3. IF 시간 충돌이 감지되면, THEN THE Schedule_Validator SHALL 충돌하는 과목 쌍과 겹치는 시간대를 명시하여 사용자에게 알린다.
4. WHEN 시간 충돌이 감지되면, THE Schedule_Validator SHALL 충돌 없는 최적 과목 조합을 추천한다.
5. IF 희망 과목 목록에서 충돌 없는 조합이 존재하지 않으면, THEN THE Schedule_Validator SHALL "모든 과목 조합에서 시간 충돌이 발생합니다. 과목을 변경해주세요." 메시지를 반환한다.
6. THE Schedule_Validator SHALL 충돌 검증 결과를 시간표 형태(요일×교시 격자)로 시각화하여 표시한다.
7. IF 희망 과목 중 Course_DB에 시간 정보가 없는 과목이 있으면, THEN THE Schedule_Validator SHALL 해당 과목명을 명시하고 "시간 정보를 찾을 수 없는 과목이 있습니다." 메시지를 반환한다.

### 요구사항 11: 데이터 적재 파이프라인

**User Story:** As a 개발팀, I want to 강의 데이터와 학사안내 원문을 자동으로 DB에 적재하고 싶다, so that 서비스 운영에 필요한 데이터를 수동 작업 없이 준비할 수 있다.

#### 인수 기준

1. THE Data_Pipeline SHALL data/courses.csv 파일을 파싱하여 DynamoDB courses 테이블에 적재한다.
2. THE Data_Pipeline SHALL courses.csv의 각 행을 강의명, 교수명, 학점, 영역 구분, 에브리타임 평점, 강의평 요약, 요일, 교시 필드로 매핑하여 저장한다.
3. THE Data_Pipeline SHALL data/data.md 파일을 800자 단위 Chunk로 분리한다.
4. THE Data_Pipeline SHALL 각 Chunk에 대해 Amazon Bedrock Titan Embeddings API를 호출하여 임베딩 벡터를 생성한다.
5. THE Data_Pipeline SHALL 생성된 Chunk 텍스트와 임베딩 벡터를 Amazon OpenSearch Serverless Vector_DB에 저장한다.
6. IF courses.csv 파일 형식이 올바르지 않으면, THEN THE Data_Pipeline SHALL 오류가 발생한 행 번호와 원인을 로그에 기록하고 해당 행을 건너뛴다.
7. IF data.md 파일이 존재하지 않으면, THEN THE Data_Pipeline SHALL "학사안내 원문 파일(data/data.md)을 찾을 수 없습니다." 오류를 반환한다.
8. IF Amazon Bedrock Titan Embeddings API 호출이 실패하면, THEN THE Data_Pipeline SHALL 실패한 Chunk를 로그에 기록하고 3회까지 재시도한다.
9. FOR ALL courses.csv의 유효한 행에 대해, 적재 후 DynamoDB에서 조회하면 원본 데이터와 동일한 값을 반환한다 (라운드트립 속성).
10. FOR ALL data.md의 Chunk에 대해, Vector_DB에 저장 후 해당 Chunk 텍스트로 유사도 검색하면 해당 Chunk가 상위 결과에 포함된다 (라운드트립 속성).

### US-04. 수강신청 난이도 산정
As a 한양대 재학생
I want to 각 강의의 수강신청 난이도를 확인하면
So that 수강신청 성공 가능성이 높은 시간표를 선택할 수 있다

수강신청 난이도 계산 공식:
  - 경쟁률 점수 (40%):
    hanyang_sugang.수강/정원 파싱
    → (현재수강 / 정원) × 100 = 경쟁률%
    → 90% 이상: 매우어려움 / 70~89%: 어려움
    → 50~69%: 보통 / 50% 미만: 쉬움
  - 평가수 점수 (30%):
    lectures.평가수 기반
    → 평가수 많을수록 인기 강의 = 난이도 높음
    → 50개 이상: 높음 / 20~49: 보통 / 20 미만: 낮음
  - 강의평 별점 (30%):
    lectures.강의평_별점 평균
    → 별점 높을수록 인기 = 난이도 높음
  최종 난이도: 상/중/하 3단계로 표시
